라이브러리 자체 실패 library self-fail

jsDelivr에서 임포트 성공, window.tournamentBracket(...) 호출. 내부적으로 라이브러리가 modern browser에서 read-only인 shadowRoot에 쓰려고 함. 게시된 빌드 1.0.7이 사양보다 오래되었거나 상위에서 이후 패치됐음 — 1.0.8이 나오거나 더 오래된 컨테이너에 wrap해 보세요. Imported successfully from jsDelivr, then called window.tournamentBracket(...). Internally, the library tries to write to a shadowRoot property that is read-only in modern browsers. Either the published build at 1.0.7 is older than the spec it targets, or upstream has since been patched in a later version — try 1.0.8 when released, or wrap in an older container.

Code that drives them all

bracket-samples.js

// 5 render functions, one per library. Each writes DOM into a target.
// Renderers are intentionally compact; the goal is to show the
// design signature of each library, not a 1:1 implementation.

function renderBracketsViewer(target) { /* minimal boxed teams */ }
function renderVue                (target) { /* column-per-round, divider */ }
function renderJqBracket          (target) { /* stage-label + grid + click hint */ }
function renderReact              (target) { /* SVG rect + L-shape connector */ }
function renderCssOnly            (target) { /* flex column with margin-top trick */ }

document.querySelectorAll('[data-sample]').forEach(el => {
  const which = el.dataset.sample;
  ({ bv: renderBracketsViewer, vue: renderVue,
     jq: renderJqBracket, react: renderReact,
     css: renderCssOnly })[which]?.(el);
});

각 렌더러는 끝까지 읽기 충분히 작습니다. 이 repo의 bracket-samples.js 파일이 5개의 셀 구현을 모두 담고 있습니다; 위에 나열된 라이브러리들은 비교 가능한 실제 upstream 레포입니다. Each renderer is small enough to read end-to-end. The bracket-samples.js file in this repo holds all five; the libraries above are real upstream repos you can compare against.
