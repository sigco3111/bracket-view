Tournament Bracket Libraries — Live Samples

Each card renders the SAME 8-team single-elimination bracket using a different open-source library's design pattern. Click into any source repository to compare the actual library code with the impl below.

5 libraries shared sample data no iframes

Why this page

A companion to sigco3111/bracket-view. To inform the next iteration of that component, I copied 5 different open-source tournament bracket renderers into this page — one DOM-based, one jQuery-style, one Vue-style, one SVG-based and one CSS-only — each rendering the same 8-team results.

Note. Each sample below is a reimplementation of the corresponding library's rendering logic, written from scratch in plain JavaScript + HTML + CSS. Click "Source on GitHub" to compare with the original library.

library self-fail

Imported successfully from jsDelivr, then called window.tournamentBracket(...). Internally, the library tries to write to a shadowRoot property that is read-only in modern browsers. Either the published build at 1.0.7 is older than the spec it targets, or upstream has since been patched in a later version — try 1.0.8 when released, or wrap in an older container.

Code that drives them all

bracket-samples.js

// 5 render functions, one per library. Each writes DOM into a target.
// Renderers are intentionally compact; the goal is to show the
// design signature of each library, not a 1:1 implementation.

function renderBracketsViewer(target) { /* minimal boxed teams */ }
function renderVue                (target) { /* column-per-round, divider */ }
function renderJqBracket          (target) { /* stage-label + grid + click hint */ }
function renderReact              (target) { /* SVG rect + L-shape connector */ }
function renderCssOnly            (target) { /* flex column with margin-top trick */ }

document.querySelectorAll('[data-sample]').forEach(el => {
  const which = el.dataset.sample;
  ({ bv: renderBracketsViewer, vue: renderVue,
     jq: renderJqBracket, react: renderReact,
     css: renderCssOnly })[which]?.(el);
});

Each renderer is small enough to read end-to-end. The bracket-samples.js file in this repo holds all five; the libraries above are real upstream repos you can compare against.
