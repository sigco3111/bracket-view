A side-by-side look at the most interesting GitHub libraries for rendering interactive single-elimination brackets. Each card below embeds an external demo iframe with a code-snippet takeaway.

Why this page exists

I am building sigco3111/bracket-view, a zero-dependency interactive bracket Web Component. To inform the next iteration, I collected the most production-quality open-source bracket libraries and put their live demos and key code patterns in one place so the trade-offs are easy to compare.

Note. Embedded iframes depend on the demo source allowing X-Frame embedding. If a card shows an opaque box, the upstream site is blocking the iframe — use the "Open demo" link below it to view it in a new tab.

Code snippets I am borrowing from each

bracketry — equal-width columns grid

/* lib/styles/main.scss */
.equal-width-columns-grid {
    display: grid;
    grid-auto-flow: column;
    grid-auto-columns: minmax(0, 1fr);
}

/* The trick: grid-auto-columns minmax(0, 1fr) forces every
 * automatically-placed column to be the same width regardless
 * of its content's intrinsic min-size. That's why bracketry
 * cards stay equal when one round has 16 cards and the next
 * has 2. */

react-tournament-brackets — SVG L-connector

/* src/components/bracket/SvgConnector.tsx */
<path
  d={`M${x1} ${y1} H${xMid} V${y2} H${x3}`}
  stroke="rgba(255,255,255,.2)"
  fill="none"
/>

// y1 = match(i).y + 0.5 * matchHeight
// y2 = match(target).y + 0.5 * matchHeight
// xMid = (x1 + x3) / 2
// This locks the curve's vertical segment to the column-gap

brackets-viewer.js — CSS variables theme

/* dist/brackets-viewer.min.css — :root vars */
:root {
  --match-spacing: 12px;
  --match-width:  200px;
  --round-margin:  50px;
  --connector-size: 0.5rem;
}

/* Every internal class namespaced under ".bracket" so the
 * theme tokens stay inside the rendered container. */
